Q1) WHY IS VOLUME NEGATIVE? DOES IT MEAN INHALING?
==================================================
Short answer:
- Negative volume does NOT automatically mean inhaling.

Correct interpretation:
- Volume in these files is usually a sensor/reference-relative signal.
- The sign (+/-) depends on baseline calibration and offset.
- Inhaling/exhaling is better inferred from TREND (slope/change over time),
  not from sign alone.

Useful rule:
- If volume is increasing over nearby samples, that suggests inhale direction.
- If volume is decreasing, that suggests exhale direction.
- Absolute sign alone can be misleading.

Evidence from this sample file:
- Volume has positive, zero, and negative values.
- Median is 0, so baseline appears centered around zero.

Q6) WHAT IS RMSE? WHAT IS MAE?
==============================
These are regression error metrics (used when predicting numeric values).

MAE (Mean Absolute Error):
- Average absolute difference between prediction and true value.
- Formula:
  MAE = (1/n) * sum(|y_true - y_pred|)
- Easy to interpret in original units.
- Less sensitive to large outliers than RMSE.

RMSE (Root Mean Squared Error):
- Square root of average squared errors.
- Formula:
  RMSE = sqrt((1/n) * sum((y_true - y_pred)^2))
- Penalizes large errors more strongly than MAE.
- Useful when big mistakes are especially costly.

How to explain quickly:
- MAE = average mistake size.
- RMSE = outlier-sensitive mistake size.
- Usually RMSE >= MAE.

In this project context:
- Current core tasks are classification (0/1 labels), so metrics like
  Accuracy/F1/Balanced Accuracy/Confusion Matrix are primary.
- RMSE/MAE become relevant if you add regression (e.g., volume forecasting).
