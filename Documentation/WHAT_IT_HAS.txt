ML PIPELINE FEATURES
--------------------

DATA LOADING
------------
✓ Unified loader for multiple formats:
  - .dat files (primary format)
  - .txt files (semicolon-separated)
  - .csv files (standard CSV)
✓ Automatic format detection
✓ Header metadata parsing (.dat/.txt files)
✓ Patient and file ID tracking
✓ Error handling for malformed files
✓ Support for large files (streaming where possible)

FEATURE ENGINEERING
-------------------
✓ Window-based feature extraction:
  - Fixed-length time windows (default: 2 seconds)
  - Configurable window size (1.0 to 5.0 seconds)
  - Non-overlapping windows
  - Minimum window size filtering

✓ Volume Statistics (7 features):
  - Mean volume
  - Standard deviation
  - Minimum value
  - Maximum value
  - Range (max - min)
  - Change (end - start)
  - Rolling mean

✓ Balloon Valve Features (2 features):
  - Fraction inflated (status = 4)
  - Fraction deflated (status = 1)

✓ Patient Switch Feature (1 feature):
  - Fraction of time switch is ON

✓ Gating Mode Feature (1 feature):
  - Fraction of time in "Automated" mode

Total: 11 features per window

LABEL EXTRACTION
----------------
✓ Breath-hold vs Free-breathing classification:
  - Based on Balloon Valve Status
  - Window-level labeling
  - Handles multiple status values

✓ Gating OK vs Not OK classification:
  - Based on Gating Mode field
  - Handles spelling variants ("Manual Overide")
  - Window-level labeling

MODEL TRAINING
--------------
✓ Multiple ML algorithms:
  - Random Forest (100 trees)
  - Logistic Regression
  - Support Vector Machine (SVM with RBF kernel)
  - XGBoost (if installed)

✓ Patient-based train/test splits:
  - Prevents data leakage
  - Configurable split ratios (default: 70/15/15)
  - Random seed for reproducibility

✓ Feature preprocessing:
  - StandardScaler (mean=0, std=1)
  - Missing value handling
  - Consistent with inference pipeline

✓ Model evaluation:
  - Accuracy
  - Balanced accuracy (used for selection)
  - F1 score
  - Confusion matrix
  - Per-model metrics saved

✓ Model persistence:
  - Best model saved as pickle file
  - Includes scaler and task metadata
  - Metrics saved as JSON
